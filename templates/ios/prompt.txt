## iOS Development Guidelines

### Architecture
- Use SwiftUI for new views
- Follow MVVM pattern: View + ViewModel
- Use Swift's Observation framework for reactive state
- Keep views focused and composable

### UI/UX
- Use SF Symbols for icons
- Support Dynamic Type for accessibility
- Add haptic feedback for user actions where appropriate
- Handle loading, error, and empty states

### Code Quality
- Use Swift's type system effectively
- Handle optionals safely (avoid force unwrapping)
- Add documentation comments for public APIs
- Follow Swift naming conventions

### Testing
- Write unit tests for ViewModels and business logic
- Test validation rules and edge cases
- Place tests in the appropriate test target
